CNAME 会被覆盖问题
搭建在 GitHub Page 上的话，GitHub 会自动给你一个二级域名，

但有些小伙伴可能像我一样，还是喜欢一级域名，于是就将自己的域名 CNAME 解析到了 Github 上。

但发现了一个小问题：

每次 hexo d 的时候， CNAME　文件会被覆盖。

对此，解决方案是：将　CNAME 文件放到本地的项目的 根目录/source 目录下。

如：

MyFirstEnglishBlog
